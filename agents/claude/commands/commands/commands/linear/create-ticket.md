## Context

You will create a new Linear issue with appropriate tags based on the type of work requested.
The team is always Engineering team
Description, Lane/status, is in the [$ARGUMENTS]

## Your task

- Create the Linear issue with a clear title, description and acceptance criteria
- Unless stated, descriptions should not be over 100 words
- Unless stated, set the status of the ticket Needs Refining 
- keep simple, non verbose, a non technical or product manager can understand
- Apply the relevant existing tags, never create a new tag
- Add any additional labels that make sense for the context but not verbose
- Leave any open questions if relevant, optional

## Available Tags

- **bug**: For defects or issues that need fixing
- **feature**: For new functionality or capabilities
- **spike**: For research or investigation tasks
- **improvement**: For enhancements to existing functionality
- **infra**: For enhancements to existing functionality

## Instructions

1. Ask for clarification on the ticket details if needed
2. Create the issue with proper formatting
3. Apply the most relevant tag(s) based on the request
4. Provide the created ticket URL for reference
